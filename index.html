Android 录制视频然后转GIF图

发表于 2017-01-17 |

最近使用Hexo搭建个人博客，开始着手整理下自己写的代码，Demo展示静态图不好体现，现在使用Android命令行进行录制视频，然后通过其他工具转为GIF，更为生动。

注:Android4.4(API19)以上的手机，命令screenrecord可以帮助我们简单实现。

录制命令

adb shell screenrecord /sdcard/videoname.mp4

/sdcard/videoname.mp4 是视频所保存的目录和名称。
输入命令按Enter后，就会开始录制180s，Ctrl+C可以提前终止。

获取视频文件

adb shell pull /sdcard/videoname.mp4

/sdcard/videoname.mp4 就是我们刚才录制存放的目录地址，此命令将SD卡中的视频文件保存到本地。

注:命令所在的目录下

视频转GIF

目前使用EZGIF 和 IMGFLIP还是相当便捷的，也有一些其他的工具，可以自己查阅。

Android 中ImageView加载超长图片会报一个警告，并且无法在ImageView上渲染图片。

OpenGLRenderer: Bitmap too large to be uploaded into a texture (440x13323, max=4096x4096)

据查，开启了硬件加速后，OpenGLRenderer能够加载Bitmap的具有限制最大值。
eg：当前的最大限制是 4096x4096
这个限制的最大值可以根据方法获得：

canvas.getMaximumBitmapHeight()和canvas.getMaximumBitmapWidth()

当ImageView无法渲染这些图片的时候，我们可以使用WebView来显示这张图片

我们需要借助一个库 jsoup，一个Java 的HTML解析器
